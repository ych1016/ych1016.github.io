Java深拷贝与浅拷贝

发表于 2025-03-24 分类于 Java 阅读次数： 本文字数： 1.7k 阅读时长 ≈ 6 分钟
Java对象的深拷贝与浅拷贝的区别与实现

一、引言

  在Java中存在这个接口 Cloneable。实现该接口的类都会具备被拷贝的能力，同时拷贝是在内存中进行的，在性能方面比直接通过new生成的对象快，特别是在大对象的生成上，使得性能的提升非常明显。

  代码实现主要是让类继承 Cloneable，然后重写 clone 方法，在重写 clone 方法时，需要注意，如果对象里面出了基本数据类型变量外， 还有其他引用类型变量，那么如果要实现深拷贝，就需要其他引用类型的类也继承 Cloneable 接口，并实现 clone 函数，然后再该类的 clone 方法中调用其他类的 clone 函数。

  在进行拷贝时，如果需要对引用变量进行深拷贝的话，则需要调用其 clone 方法。

二、深拷贝与浅拷贝的区别

  浅拷贝：对基本数据类型进行值传递，对引用数据类型进行引用的拷贝，实际就是拷贝指向实际内存的引用，并不会对实际内存的值进行拷贝。

只是增加了一个指针指向已存在的内存地址

  深拷贝：对基本数据类型进行值传递，对引用数据类型，会创建一个新的对象，并复制原有对象的实际值。

增加了一个指针并且申请了一个新的内存，使这个增加的指针指向这个新的内存。

  深拷贝与浅拷贝最根本的区别在于：是否真正获取一个对象的复制实体，而不是引用。

三、代码实现

1、拷贝引入

（1）引用拷贝

  实际上的操作是：Java虚拟机会创建一个指向对象的引用变量的拷贝

（2）对象拷贝

  实际上的操作是，会创建对象本身的一个副本

（3）浅拷贝

  在拷贝时，对于调用了 clone 方法的类，实行深拷贝，对于直接赋值 （使用 = ），则实行浅拷贝。在这个栗子中，学生类有重写 clone 方法。但是 clone 方法里却没有对其内部的teacher引用实行深拷贝（即调用teacher的clone方法）， 所以会导致学生类进行深拷贝了，teacher对象只是简单的浅拷贝。

（3）深拷贝

  深拷贝是一个整个独立的对象拷贝，深拷贝会拷贝所有的属性,并拷贝属性指向的动态分配的内存。当对象和它所引用的对象一起拷贝时即发生深拷贝。深拷贝相比于浅拷贝速度较慢并且花销较大。 简而言之，深拷贝把要复制的对象所引用的对象都复制了一遍。

  在下面的这个栗子中，由于学生类在实现clone方法时，也调用了teacher的clone方法，所以其内部对应的teacher也会进行深拷贝

四、序列化深拷贝

1、引言

  若系统中存在大量的对象都是通过拷贝生成的，如果每一个类都写一个 clone() 方法，并将还需要进行深拷贝，新建大量的对象。这个工程是非常巨大的，这里可以采用序列化的方式来实现对象的拷贝。

2、定义

  利用序列化来做深复制,把对象写到流里的过程是序列化（Serilization）过程，而把对象从流中读出来的过程则叫做反序列化（Deserialization）过程。 应当指出的是，写在流里的是对象的一个拷贝，而原对象仍然存在于JVM里面。 利用这个特性，可以做深拷贝 。并且该新对象与母对象之间并不存在引用共享的问题，真正实现对象的深拷贝。

  同时，需要注意的是，要想使用序列化进行深拷贝，则对应的对象必须要实现Serializable接口，否则是没有办法实现克隆的。
